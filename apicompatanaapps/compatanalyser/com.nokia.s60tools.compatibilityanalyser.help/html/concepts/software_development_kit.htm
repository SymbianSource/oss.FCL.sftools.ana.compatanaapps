Software Development Kit (SDK)

To analyse compatibility, you must have a Symbian based Software Development Kit (SDK) installed on your workstation. An SDK contains the tools, documentation and code examples that support the development of applications.

To configure SDKs in Carbide.c++, go to Windows > Preferences > Carbide.c++ > SDK Preferences > Add New SDK, and select the SDK to be added.

Baseline SDK and Current SDK

To analyse possible compatibility breaks between different versions of header and library files, two types of SDKs should be defined:

  • Baseline SDK: The SDK that is used to build/compile the given header and library files. The version number of this SDK must be less than the current SDK.
  • Current release SDK: The SDK that is used to build/compile the given current header and library files. The version number of this SDK must be greater than the baseline SDK.
