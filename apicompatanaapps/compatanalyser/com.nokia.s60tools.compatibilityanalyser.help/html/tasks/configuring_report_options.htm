Configuring report file options

When Compatibility Analyser runs an analysis, you can specify the name and location for the report, as well as whether or not it is filtered against a knownissues.xml file.

To configure the report options, do the following:

  1. Type in a file name for the report. By default, the reports are named with the timestamp of the operation.
  2. Click Browse to select where to create the reports.
  3. With the filtering options, you can either:
    • Select No to generate the report without filtering.
    • Select Yes and click Configure Knownissues file. See Configuring the knownissues.xml file for instructions.
  4. Do either of the following:
    • To start the analysis, click Finish. See Viewing the reports for the next steps.
    • To save the settings of this analysis into a file for later use, click Next. See Saving the analysis into a configuration file for instructions.
