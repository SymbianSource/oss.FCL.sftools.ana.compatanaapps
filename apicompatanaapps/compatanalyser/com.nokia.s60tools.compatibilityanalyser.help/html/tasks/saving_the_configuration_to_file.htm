Saving the analysis into a configuration file

Before you start

When Compatibility Analyser analyses a project, you can save the settings into a configuration file for further use. The file contains the same values that were specified on the wizard (for example, baseline and current SDKs, toolchains, and the knownissues file). Files with this extension are associated with Compatibility Analyser by default and be used to launch the analysis.

Tip: Create a configuration file for, for example, different projects or releases you plan to test.

To save the configuration:

  1. On the Configuration Information wizard page, select the check box.
  2. Click Browse to locate the project to which the file belongs. Default is the currently active project.
  3. Enter a filename for the configuration.
  4. Click Finish to start the analysis. See Viewing the reports for the next steps.

Running an analysis using a saved configuration

To launch a saved configuration on the Project Explorer view, do either of the following:

  • Right-click the .comptanalyser file of your choice and select Open With... > Run static analysis with Compatibility Analyser.
  • Double-click the .comptanalyser file of your choice.

Note: Analysis starts immediately without the wizard and the previous result files are overwritten.

Modifying a saved configuration

To modify a saved configuration (for example, to change the location of the result file or analysed components), do the following:

  1. Select the .comptanalyser file you want to modify and select File > Open With... > Edit Compatibility Analyser Configuration File...

    The Compatibility Analyser wizard is launched.

  2. On the wizard pages, modify the values as needed.
  3. To save the changes, click Finish.

Note: Modification of configuration is encouraged only in the above specified way. Manual editing is not encouraged.
