Configuring the knownissues.xml file

Before you start

When checking for Binary Compatibility (BC), Compatibility Analyser produces a report that lists those issues that may affect BC. You can filter these reports so that they compare the found issues against a knownissues.xml file. This file contains information on whether or not the same BC problems have occurred before in the Symbian platform and what should be done about them.

Note: If the knownissues.xml file does not return any information, the BC issue in question is new and introduced by the product. In this case, consult a Binary Compatibility expert who can carry out further analysis.

To configure this file, do the following:

  1. In the Knownissues dialog, select any of the following options:
    • Use the default knownissues.xml files supplied with the tool.
    • Use a knownissues.xml file from a Web server by defining an URL.

      Tip: This option ensures that the latest version of this file is always used and the compatibility reports do not list known issues that are already fixed.

    • Use a list of knownissues.xml files from a Web server by defining an URL.
    • Browse for knownissues.xml files from your local file system.
  2. Click OK.
