Messages and notifications

The following table lists the messages that may be displayed when analysing files and filtering reports.

Message Explanation Action

Select at least one type of analysis (headers and/or libraries).
Version string cannot be null

The Symbian SDK version list box is empty.

Select or type in the version number of the selected SDK in the product SDK version list box.
Root directory can not be null

On the headers page, the directory path is empty under the header selection group.

Type in or browse for the directory path of header files.
No header files are selected. Select files to be analysed.

In the headers analysis page, the header file list is empty.

Add headers using the Add files button.

Select header types to be analysed.

On the header analysis page, nothing is selected under the Selection of file types to be analysed group.

Select file types from the four options (.h , .hrh , .mbg , .rsg).

Selected types do not match with the selected header files.

Files added to the headers list in the header analysis page do not match with the file types selected.

For example, .mbg or.rsg files cannot be selected for header analysis.

Remove all the files from the list or add only the files of the selected type. In addition, you can also check types of files in the list.
No platform header paths are defined. This may cause compilation errors.

On the platform headers page, the list of system include paths is empty.

Add system include paths using the Add includes button.

No library files are selected for analysis.

On the library analysis page, the library list is empty.

Add library files using the Add files button.
No Baseline Profiles are Selected.

Click the Configure Baseline profile button to start creating a profile.
Report FileName can not be null.

Type any name into the text field.
Report File Path can not be null.

Provide the directory path using the Browse button.
Add minimum one report file.

In the Report Filter Wizard, the list of report files is empty.

Add report files to be filtered.
